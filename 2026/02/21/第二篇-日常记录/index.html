切换站点（新站 / 旧站）

切换发布内容在 GitHub 网页端完成：

1.打开仓库：Settings → Pages

2.Source 选择 Deploy from a branch

3.Branch 选择：

    gh-pages：发布 Hexo 新站（当前）

    legacy：发布旧站备份

保存后等待片刻刷新即可。

更新日志（里程碑）

v0（旧站阶段）：从零搭建静态个人站（HTML/CSS/JS），并上线到 GitHub Pages

v0.1：加入动画、主题切换、可复用导航组件；实现博客页面与文章结构

v0.2：博客支持 Markdown 渲染、文章路由与列表自动生成；处理缓存与部署细节

v1（Hexo 迁移）：迁移到 Hexo 架构；采用 main（源码）+ gh-pages（发布）分离；保留 legacy 作为旧站备份；新站成功发布到 https://sicci-zzf.github.io

备注

node_modules/、public/、.deploy_git/、db.json 均已忽略，不纳入版本控制

如遇更新后页面未变化，可等待 Pages 部署完成并强制刷新（⌘⇧R）

你怎么保存它

在终端执行：

把上面的内容粘贴进去保存，然后提交：

git add README.md
git commit -m "docs: add README"
git push
